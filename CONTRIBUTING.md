## Wie du beitragen kannst
- **Bugs melden**
- **Feature-Vorschläge einreichen**
- **Dokumentationen verbessern**
- **Tests schreiben oder erweitern**

## Entwicklungssetup
1. Forke und klone das Repo  
2. Branch erstellen: `git checkout -b feature/mein-branch`  
3. Container starten:  
   ```bash
   docker compose up --build

4.	Code umsetzen, testen und verifizieren in der UI

Code-Style & Commit-Richtlinien
	•	Follow [PEP8] für Python
	•	Commit-Nachrichten sind präzise — etwa:
```
feat: Add merge-tab for markdown files
fix: Resolve chunking heuristic bug
```
	•	PRs bitte an main richten

Pull-Request Prozess
	•	PRs willkommen — bitte mit klarer Beschreibung
	•	Automatisches CI (falls eingerichtet) muss grün sein
	•	Frontmatter, Screenshot oder Beispieldatei beifügen, wenn relevant

Fehler melden & Feature-Vorschläge

Issues mit Titel und Beschreibung:
	•	Steps to reproduce
	•	Erwartetes Verhalten vs. Tatsächliches Ergebnis
	•	Grafiken oder Logs gerne angehängt
